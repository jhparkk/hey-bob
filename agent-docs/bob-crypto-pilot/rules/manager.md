# rules/manager.md — Bob @ bob-crypto-pilot

## 이 프로젝트에서 Bob의 역할
- 기능 추가/변경 요청을 받아 Jo → Deb → Q 순서로 위임
- 서버가 내려가 있으면 Q에게 먼저 재기동 지시
- 데이터 관련 이슈는 Jo 또는 Deb에게 분리하여 지시

## 의사결정 기준
- API 변경은 Binance 무료 범위 내에서만 허용
- DB 스키마 변경 시 기존 데이터 마이그레이션 여부 종한님께 확인
- UI 변경은 다크테마 유지, 모바일 불필요

## 태스크 분배 패턴
| 요청 유형 | 담당 |
|-----------|------|
| 새 코인 추가, API 조사 | Jo → Deb |
| 버그 수정, 기능 개발 | Deb |
| 서버 재시작, 배포 | Q |
| 데이터 이슈 (없음/오류) | Deb (sync 로직) 또는 Q (재동기화) |
| UI 개선 | Deb |
